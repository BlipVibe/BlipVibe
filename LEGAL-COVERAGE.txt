TECHNICAL SECURITY MEASURES
=========================================================================

XSS PROTECTION
- escapeHtml() applied to 45+ user-content injection points
- Covers: post content, display names, bios, comments, messages,
  group names/descriptions, notifications, link previews, album titles

COOKIE CONSENT
- All third-party embeds gated behind cookie consent
- YouTube, Vimeo, Spotify, SoundCloud (iframes)
- TikTok, Twitter/X, Instagram (SDK scripts / iframes)
- Placeholder UI shown when blocked
- Manage cookies in Settings (accept/revoke)

PROFILE DATA PROTECTION
- Sensitive columns (skin_data, birthday, email) revoked from
  authenticated/anon SELECT
- SECURITY DEFINER RPC (get_own_profile) for own data access
- Column-level Postgres grants

ACCOUNT DELETION
- Delete My Account button in Settings (double confirmation)
- Cleans up storage files (avatars, post images)
- delete_own_account RPC deletes profile + auth.users record
- CASCADE deletes all posts, comments, likes, follows, etc.
- profiles_delete RLS policy for user self-deletion

GEOLOCATION
- Off by default, opt-in only
- Only fires when settings.showLocation is true
- Approximate area only, no precise coordinates stored

SEARCH INJECTION PREVENTION
- Server-side search_profiles() RPC with parameterized ILIKE
  (no client-side string interpolation into PostgREST filters)
- Sanitizes LIKE wildcards (%, _, \) server-side
- sbSearchProfiles() calls RPC instead of .or() filter

ADMIN SECURITY
- is_admin column cannot be set by client (REVOKE UPDATE)
- All admin RPCs check is_admin server-side (SECURITY DEFINER)
- Cannot delete/suspend other admin accounts
- Admin actions: view users, suspend, delete (with confirmation)
- Admin action logging: admin_logs table records all admin
  suspend/unsuspend/delete actions with timestamps and details

RATE LIMITING
- Auth (login/signup/password reset): Frontend cooldown helper
  prevents rapid-fire attempts (3s/5s/10s). HTTP 429 handling.
- Direct messages: Server-side RPC limits to 30 messages per
  60 seconds per sender
- Post creation: 5-second frontend cooldown between posts

FILE UPLOAD HARDENING
- MIME whitelist: JPEG, PNG, WebP, GIF only
- Blocked types: SVG, HTML, JavaScript (XSS vectors)
- Size limits: avatars/covers 5MB, post images 10MB
- Extension-MIME match validation
- Applied to all upload paths (avatar, cover, post, background,
  message image)

ROW LEVEL SECURITY (RLS) POLICIES
- notifications: users can delete their own
- messages: users can delete messages they sent
- user_skins: users can delete their own
- group_skins: group owners can delete
- storage.objects (posts bucket): authenticated users can delete
